# Task Service

HTTP API на Go для управления задачами медицинского трекера.

Сервис поддерживает:

- разовые задачи
- повторяющиеся шаблоны задач
- автоматическое создание ближайших экземпляров повторяющихся задач в трекере

Поддерживаемые типы повторяемости:

- `daily` - каждый `n`-й день
- `monthly` - каждый месяц в выбранное число от `1` до `30`
- `specific_dates` - только на выбранные даты
- `odd_days` / `even_days` - только по нечётным или чётным дням месяца

## Требования

- Go `1.23+`
- Docker и Docker Compose

## Быстрый запуск

```bash
docker compose down -v
docker compose up --build
```

После старта сервис доступен по адресу [http://localhost:8080](http://localhost:8080).

## Swagger

- UI: `http://localhost:8080/swagger/`
- OpenAPI JSON: `http://localhost:8080/swagger/openapi.json`

## Основные маршруты

Базовый префикс API: `/api/v1`

- `POST /tasks`
- `GET /tasks`
- `GET /tasks?include_templates=true`
- `GET /tasks/{id}`
- `PUT /tasks/{id}`
- `DELETE /tasks/{id}`

## Как теперь работает повторяемость

Система разделяет:

1. **Шаблон повторяющейся задачи**
2. **Обычные задачи-экземпляры**, которые реально показываются в трекере

По умолчанию `GET /tasks` возвращает только рабочие задачи, которые пользователь должен видеть в трекере:

- разовые задачи
- сгенерированные экземпляры повторяющихся задач

Шаблоны скрыты из обычного списка, чтобы не засорять трекер. Если нужно увидеть и шаблоны тоже, используйте:

```text
GET /api/v1/tasks?include_templates=true
```

При создании или чтении списка сервис лениво догенерирует ближайшие экземпляры повторяющихся задач на окно в **30 дней вперёд**. Это решает сразу две проблемы:

- список задач не раздувается на сотни записей на год вперёд
- трекер всё равно получает реальные задачи по заданному правилу

## Примеры запросов

### Разовая задача

```json
{
  "title": "Провести обход пациентов",
  "description": "Утренний обход в терапевтическом отделении",
  "status": "new",
  "scheduled_at": "2026-04-21T09:00:00Z"
}
```

### Повторяющийся шаблон - каждый второй день

```json
{
  "title": "Обзвон пациентов",
  "description": "Контроль самочувствия после выписки",
  "status": "new",
  "scheduled_at": "2026-04-21T10:00:00Z",
  "recurrence": {
    "type": "daily",
    "every_n_days": 2
  }
}
```

### Повторяющийся шаблон - только на конкретные даты

```json
{
  "title": "Инвентаризация",
  "description": "Проверка остатков расходников",
  "status": "new",
  "scheduled_at": "2026-04-21T08:00:00Z",
  "recurrence": {
    "type": "specific_dates",
    "dates": ["2026-04-24", "2026-04-30"]
  }
}
```

`scheduled_at` в этом случае всё равно участвует как первая дата серии.
