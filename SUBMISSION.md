## Что реализовано

В проект добавлена поддержка периодических задач.

Поддержаны типы повторяемости из задания:

- ежедневные задачи с интервалом `каждый n-й день`;
- ежемесячные задачи на выбранное число месяца (`1-30`);
- задачи на конкретные даты;
- задачи по четным / нечетным дням месяца.

## Принятое решение

Периодическая задача хранится как шаблон серии.
На его основе сервис автоматически создает рабочие экземпляры задач в трекере на ближайшее окно вперед.

Такой подход позволяет:

- не засорять основной список задач шаблонами;
- сохранять обычный CRUD API для рабочих задач;
- получать реальные задачи, с которыми пользователь дальше работает в интерфейсе;
- контролировать объем автоматически создаваемых записей.

По умолчанию `GET /api/v1/tasks` возвращает только рабочие задачи.
Для просмотра шаблонов можно использовать `GET /api/v1/tasks?include_templates=true`.
